We are the core development team behind **Young Folks LV**, a dynamic youth organization based in Latvia focused on non-formal education, community building, and empowering the next generation. 

Our mission on GitHub is to build, maintain, and scale the digital infrastructure that connects thousands of young people with educational courses, travel opportunities, and community projects.

---

## 🚀 The Project: Next-Gen Web Platform

We are currently transitioning from a legacy CMS to a fully custom, highly scalable, and interactive web application built from scratch. This unified platform will serve as the central hub for all Young Folks activities.

### Key Features Under Development:
*   **🌍 Multi-Branch Ecosystem:** Dedicated sections for different age groups (Kids, Teens, Youth, Forever Young) and directions (Travel, Projects, Startups).
*   **🛍️ Custom E-Commerce:** An integrated shop for exclusive, customizable Young Folks merchandise.
*   **💳 Donation System:** Seamless Stripe integration to support our non-profit initiatives and non-formal education programs.
*   **⚙️ Custom CMS Panel:** A powerful admin dashboard (CRUD) to manage courses, news, applications, and user data.

## 💻 Our Tech Stack

We leverage modern, type-safe technologies to ensure performance, maintainability, and an excellent developer experience.

### Frontend
*   **Framework:** React + Vite
*   **Language:** TypeScript
*   **State Management:** Zustand
*   **Routing:** React Router

### Backend & Database
*   **Framework:** Nest.js
*   **Language:** TypeScript
*   **Database:** PostgreSQL (with Prisma/TypeORM)
*   **Integrations:** Stripe API (Payments & Donations)

## 🤝 Workflow & Contribution Guidelines

Since we are a fast-moving, agile team, we follow a strict but lightweight Git workflow to keep our `main` branch stable:

1.  **Branching:** Create feature branches from `main` (e.g., `feat/auth-system`, `fix/navbar-mobile`, `chore/dependencies`).
2.  **Commits:** Use conventional commits for clarity (e.g., `feat: add Stripe webhook implementation`).
3.  **Pull Requests:** All changes must go through a PR. Code review is mandatory before merging into the main branch.
4.  **Sync:** Always pull the latest changes from `main` and resolve conflicts before requesting a review.
